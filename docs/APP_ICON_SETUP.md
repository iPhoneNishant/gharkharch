# App Icon Setup Guide

This guide will help you set up app icons for Gharkharch.

## Required Icon Files

You need to prepare the following icon files:

### 1. Main Icon (`assets/icon.png`)
- **Size**: 1024x1024 pixels
- **Format**: PNG
- **Usage**: iOS app icon and general app icon
- **Requirements**: 
  - Square image
  - No transparency (use solid background)
  - Should work well at small sizes

### 2. Adaptive Icon Foreground (`assets/adaptive-icon.png`)
- **Size**: 1024x1024 pixels
- **Format**: PNG
- **Usage**: Android adaptive icon foreground
- **Requirements**:
  - Square image
  - Important content should be centered
  - Safe area: Keep important content within 512x512 center area (to avoid being cropped on different devices)
  - Can have transparency

### 3. Splash Icon (`assets/splash-icon.png`)
- **Size**: 1024x1024 pixels (or larger)
- **Format**: PNG
- **Usage**: Splash screen logo
- **Requirements**:
  - Can be the same as main icon or a simplified version
  - Should look good on white background (as per app.json)

### 4. Favicon (`assets/favicon.png`)
- **Size**: 48x48 pixels (or 16x16, 32x32, 96x96)
- **Format**: PNG or ICO
- **Usage**: Web favicon
- **Requirements**:
  - Small size, simple design
  - Should be recognizable at very small sizes

## Steps to Replace Icons

1. **Prepare your icon images** with the specifications above
2. **Replace the files** in the `assets/` folder:
   - `icon.png`
   - `adaptive-icon.png`
   - `splash-icon.png`
   - `favicon.png`

3. **For Android**, the icons will be automatically generated by Expo/EAS Build
4. **For iOS**, the icons will be automatically generated by Expo/EAS Build

## After Replacing Icons

1. **Test locally** (if using Expo Go, icons won't update - you need a development build)
2. **Rebuild the app** using EAS Build:
   ```bash
   eas build --platform android
   eas build --platform ios
   ```

## Notes

- Icons are automatically generated for different densities by Expo/EAS Build
- For Android adaptive icons, the background color is set to white in `app.json`
- For iOS, the icon will be automatically sized for different devices
- Make sure your icons follow platform guidelines (no text that's too small, recognizable at small sizes)
